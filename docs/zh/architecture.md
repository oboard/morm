# 架构设计

`morm` 将能力拆分为三个层次：

- 代码生成层：解析实体与 mapper 注解，生成普通 MoonBit 代码
- 查询构建层：表达查询语义，不直接依赖具体数据库方言
- 引擎执行层：负责 SQL 渲染、参数绑定、执行与结果返回

## 设计原则

- 无运行时反射
- 类型驱动空值语义
- 显式引擎边界
- 优先可读与可调试

## 为什么这样设计

这样可以让问题定位更直接：生成问题看生成文件，SQL 问题看引擎渲染，数据映射问题看实体与 mapper。
